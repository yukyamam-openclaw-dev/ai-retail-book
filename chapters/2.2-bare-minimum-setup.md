# 2.2 最小限の準備で環境を構築する

「AI店長」を導入しようと思ったとき、最初にぶつかる壁が「何を用意すればいいのか？」という不安です。エンジニアではない店主の方にとって、サーバー設定や複雑なインストール作業はハードルが高く感じられるでしょう。

しかし、安心してください。現在のAIエコシステムは驚くほどシンプルに進化しています。ここでは、複雑な理論は抜きにして、「とりあえずこれを揃えれば動き出す」という最小限の構成（ベアミニマム・セットアップ）を具体的に解説します。

## 1. 必要な「道具」を揃える

AI店長を動かすには、大きく分けて「計算能力を持つハードウェア」と「AIを制御するソフトウェア」の2つが必要です。

### ハードウェアの選択肢
前章で触れた通り、Macを使うかVPS（仮想専用サーバー）を使うかで準備が変わりますが、共通して重要なのは**「メモリ（RAM）」**です。

- **Mac (Apple Silicon) の場合**: M1/M2/M3チップ搭載モデルであれば、メモリ16GB以上を推奨します。Apple Siliconの「ユニファイドメモリ」はAI処理との相性が非常に良く、家庭用PCでありながら強力なAIサーバーとして機能します。
- **VPS (クラウド) の場合**: Ubuntu 22.04 LTSなどのLinux環境を用意してください。CPUは2コア以上、メモリは最低8GB（できれば16GB以上）のプランを選んでください。

### おすすめのソフトウェア構成
複雑な設定を避けるため、以下の「三種の神器」を導入します。

1. **Ollama (オラマ)**: AIモデルを簡単に動かすためのエンジンです。一行のコマンドでAIをインストールでき、管理も簡単です。
2. **Hermes Agent**: AIに「自律性」を与えるオーケストレーション層です。単なるチャットAIを、「店長」という役割を持つエージェントへと進化させます。
3. **Docker (オプション)**: 環境を汚さず、後で簡単に移行したい場合に便利です。

## 2. AI店長の「脳」を選ぶ

AI店長にどのモデル（脳）を使わせるかは重要ですが、最初から最高性能の巨大なモデルを選ぶ必要はありません。小売店の日常業務（在庫確認やトレンド分析）であれば、軽量で効率的なモデルで十分に機能します。

- **おすすめモデル**: Llama 3 や Gemma など、8B（80億パラメータ）程度のモデルがバランス良く、メモリ消費も抑えられます。
- **運用のコツ**: 複雑な「追加学習（ファインチューニング）」は不要です。店独自のルールや商品知識は、メモ帳（Markdownファイル）やNotionなどの外部知識を持たせる「RAG（検索拡張生成）」という手法で対応します。これにより、店主がメモを書き換えるだけで、AI店長の知識を即座に更新できます。

## 3. 最速導入ロードマップ

迷わず進めるための4ステップです。

1. **Ollamaをインストールする**: 公式サイトからインストーラーをダウンロードし、起動します。
2. **ベースモデルをダウンロードする**: `ollama run llama3` などのコマンドを打ち、AIが応答することを確認します。
3. **Hermes Agentをセットアップする**: Python環境を整え、エージェントを起動します。
4. **「店のルール」を教え込む**: メモリ機能に「営業時間は〇時〜〇時」「接客は丁寧語で」といった基本ルールを書き込みます。

これで、あなたの店に24時間365日働く「デジタル店長」の土台が完成しました。次章では、この土台の上にどのように「店長としての知能」を積み上げていくかを見ていきましょう。

### 🌍 World Wide Insight: エッジAIによる「店舗の自律化」
世界的に見ると、AIをクラウドではなく店舗内（エッジ）で完結させる動きが加速しています。これはデータのプライバシー保護だけでなく、インターネット断接続時でも店を回し続けるためです。特に欧州の小規模小売店では、省電力な小型サーバーにローカルLLMを搭載し、在庫管理を完全に自動化する事例が増えています。
