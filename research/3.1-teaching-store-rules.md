# Research: 3.1 Teaching Store Rules (Memory/Skills)

## Key Concepts in Hermes Agent
Hermes Agent distinguishes between **Memory** and **Skills** to achieve high autonomy.

### 1. Memory (The 'What')
- **Definition**: Durable, declarative facts stored across sessions.
- **Retail Application**:
    - Store-specific rules (e.g., "Discount period is every Tuesday").
    - Customer preferences (e.g., "Mr. Tanaka prefers organic coffee").
    - Environmental constraints (e.g., "The warehouse lock code is 1234").
- **Value**: Prevents the AI from forgetting critical context and reduces the need for the user to repeat information.

### 2. Skills (The 'How')
- **Definition**: Reusable, procedural workflows (often defined in `SKILL.md` files).
- **Retail Application**:
    - `inventory-check`: A set of steps to query a DB and identify low-stock items.
    - `vendor-communication`: A workflow to draft and send emails to specific suppliers.
    - `promotional-update`: Steps to update SNS and in-store signage for a new sale.
- **Value**: Standardizes complex tasks and ensures accuracy (deterministic steps).

### 3. Synergy for the "AI Store Manager"
- **Logic**: When the Agent is tasked with "Update the coffee order," it uses **Memory** to remember which bean is preferred and **Skills** to execute the order process.
- **Autonomy**: This combination allows the agent to act without constant prompting, moving from a "Chatbot" to a "Manager."
